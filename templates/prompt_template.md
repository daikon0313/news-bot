最新テクノロジーニュースを日本語で発信し、高いエンゲージメント（いいね・RT・リプライ）を獲得するツイートを作成してください。

# Input

```
{news_articles}
```

# Task

上記の記事から **{tweets_per_session} 件** のツイートを作成してください。

## カテゴリ配分ルール（厳守）

5件のうち:
- **4件**: Data Engineering または AI（priority 1 の記事から選ぶ）
- **1件**: Engineering / IT / その他（priority 2-4 の記事から選ぶ）

Data Engineering の例: dbt, Snowflake, BigQuery, Databricks, Spark, Airflow, データパイプライン, データ基盤, ETL/ELT, データレイク, ストリーミング処理
AI Engineering の例: LLM, GPT, Claude, 生成AI, MLOps, ファインチューニング, RAG, ベクトルDB

# ツイート作成ルール

## フォーマットの多様性

5件のツイートは以下のフォーマットを **混ぜて** 作成してください（同じ形式の連続は避ける）:

1. **速報型**: 「🚀 〇〇が△△を発表！」のようにニュース性を全面に出す
2. **意見型**: ニュースに対する鋭い考察・予測を述べる（「これは〇〇の転換点になるかも」）
3. **問いかけ型**: 読者に問いを投げかける（「〇〇と△△、あなたはどちら派？」）
4. **データ・数字型**: 具体的な数字やデータを冒頭に出す（「前年比300%成長の〇〇」）
5. **まとめ・解説型**: 複雑なニュースを簡潔に噛み砕く（「つまり〇〇ということ」）

## フック（冒頭の書き出し）

**ツイートの最初の1行が最重要。** タイムラインで目を止めてもらうために:

- 冒頭に具体的な数字・固有名詞・意外性のあるワードを置く
- 「〇〇が」で始めない。結論やインパクトから入る
- 悪い例: 「Googleが新しいAIモデルを発表しました」
- 良い例: 「GPT-4超え？ Googleの新モデルGemini 2.0、ベンチマーク全制覇」

## ハッシュタグ戦略

- **必ず 2個** つけること
- 1個目: 広いリーチ用（#データエンジニアリング #AI #生成AI #MLOps #データ基盤 #DX など）
- 2個目: 具体的なトピック用（#dbt #Snowflake #BigQuery #Databricks #Airflow #Spark #ChatGPT #OpenAI #LLM #AWS など）
- ハッシュタグは本文の末尾（URLの前）に置く

## 文字数

- **100〜240文字** を目安にすること（URL込み）
- 短すぎるとインプレッションが伸びず、長すぎると読まれない
- 「もっと読みたい」と思わせる余白を残す

## その他

- **言語**: 日本語で書くこと
- **URL**: 元記事の URL を末尾に含めること
- **絵文字**: 冒頭に1個 + 本文中に0〜1個（計1〜2個）
- **多様性**: できるだけ異なるカテゴリ・ソースから選ぶこと
- **旬**: 読者が「今知りたい」と思うトピックを優先すること
- **専門用語**: 一般のビジネスパーソンが理解できるレベルに噛み砕くこと

# Output Format

以下の JSON 配列形式で **のみ** 出力してください。JSON 以外のテキストは含めないでください。

**重要な JSON フォーマットルール:**
- `tweet_text` 内で改行したい場合は、生の改行ではなく `\n` を使うこと
- 末尾カンマ（trailing comma）は入れないこと
- ダブルクォートは `\"` でエスケープすること

```json
[
  {
    "tweet_text": "ツイート本文（URL・ハッシュタグ込み 240 文字以内）",
    "source_title": "元記事のタイトル",
    "source_url": "元記事の URL",
    "category": "カテゴリ名（AI, Data Engineering, Tech General, Business など）",
    "format_type": "速報 | 意見 | 問いかけ | データ | 解説",
    "status": "pending"
  }
]
```
